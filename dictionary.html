calendar.csv

  • course: course name.
  • course_number: numeric ID of course, same as course_data.csv and student_data.csv.
  • module_number: numeric ID of module; IDs are ordered in sequence of modules as they enter the course.
  • release_date: module release date. Assume 00:00.00 as release time.
  • due_date: Assume 23:59:59 as due time.

course_data.csv

  • course_number: numeric ID of course.
  • module_number: numeric_ID of module, same as calendar.csv and studen_data.csv.
  • lesson_number: numeric ID of lesson; unique within each course; lesson numeric IDs are ordered in sequence within each module but may have gaps in numbering; IDs are same as in calendar.csv and student_data.csv.
  • concept_number: numeric ID of concept; unique within each course; concept numeric IDs are ordered in sequence within each lesson but may have gaps in numbering; IDs are same as in calendar.csv and student_data.csv.
  • te_number: numeric ID of teaching element; unique within each course; teaching element numeric IDs are ordered in sequence within each concept but may have gaps in numbering; IDs are same as in calendar.csv and student_data.csv.

student_data.csv

  • student_id: numeric ID of a student. Order of student numeric IDs does not matter.
  • course_number: numeric ID of course.
  • module_number: numeric ID of module.
  • lesson_number: numeric ID of lesson.
  • concept_number: numeric ID of concept.
  • te_number: numeric ID of teaching element.
  • completiondate: historic date and time of content element completion by student.

Notes

Student completion data is not fully consistent with course structure data. In some instances, you may observe that a student moved straight to the next module without registering completion of a few final elements of the previous module. This is not possible on the platform -- all content from a previous module must be completed before proceeding to the next module. In such cases, assume that the data didn't capture the timing of completion, and that the student completed all content from the previous module before starting on the next one.
